// Overview

What is milk?

milk provides a framework and tools for image processing and analysis, geared towards high-performance computing, modularity, and real-time execution pipelines.

⌨️

Command Line Interface

A powerful CLI for easy access to functions and arguments. Runtime loading of modules to extend capabilities without recompilation.

🔗

Interprocess Comm.

IPC for sharing process status, function parameters, and images across concurrent processes with low latency.

⚙️

Modular Architecture

Custom functions integrate seamlessly into the CLI. Plugin system enables composable, reusable processing units.

🐍

Python Bindings

pyMilk bindings let you interact with milk streams and data structures directly from Python workflows.

🐳

Docker Support

Official Docker image for reproducible deployments. CI/CD integration via GitHub Actions on every commit.

📡

Stream Processing

Low-latency shared memory streams enable real-time image data pipelines with TCP/UDP network transport.

// Core Data Structures

Three pillars

milk's core design is built around three interoperable data structures that together enable the full pipeline lifecycle.

01
Image Streams (streams)
Low-latency shared memory images enabling interprocess communication of image data in real time. Supports TCP/UDP network transport, semaphore-based triggering, and logging.
02
Function Parameter Structures (FPS)
Read/write function parameters at runtime through shared memory. Enables parameter sweeping, live tuning, and structured control of running processes without recompilation.
03
Process Info & Status (procinfo)
Monitor and control running processes. Tracks CPU affinity, timing statistics, loop status, and signal handling. Integrates with the TUI process manager.
// Installation

Get up and running

Check the Dockerfile for the full list of required packages.

1

Clone the repository

Download the source code from GitHub.

git clone https://github.com/milk-org/milk.git
cd milk
2

Compile (standard)

Standard CMake build.

mkdir _build
cd _build
cmake ..
make
sudo make install
3

Compile with Python module

Use the helper script for a local install with Python bindings.

# Install to a local prefix directory
./compile.sh $PWD/local
4

Set environment variables

Add these to your .bashrc or equivalent shell config.

Variable Description Example
MILK_ROOT Source code directory /home/user/src/milk
MILK_INSTALLDIR Installation directory /usr/local/milk
MILK_SHM_DIR Shared memory directory /milk/shm
5

Verify & Tutorial

Check your install, then launch the interactive tutorial (requires tmux, nnn).

milk-check       # verify installation from any directory
milk-tutorial   # interactive tutorial
// Plugins

Extend with plugins

Fetch and compile optional plugin suites to add domain-specific capabilities.

cacao

Compute And Control for Adaptive Optics. Full AO control loop pipeline.

coffee

Additional processing modules for wavefront sensing and correction.

atmosphere

Atmospheric turbulence simulation and profiling tools.

milk-extra

FFT, image filters, linear algebra, PSF tools, and more included out-of-tree.

# Fetch and compile with cacao plugins
./fetch_cacao_dev.sh
./compile.sh $PWD/local

# Fetch and compile with coffee plugins
./fetch_coffee_dev.sh
./compile.sh $PWD/local
